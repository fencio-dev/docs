## Overview

AARM is an open specification designed to enable a competitive market for AI runtime security. This page recognizes companies building in this space.

<Note>
**Want to be listed?** If you're building in this space, [open a GitHub issue](https://github.com/aarm-spec/aarm/issues) with your company details.
</Note>

---

## Conformant vs Aligned

Companies can be listed under two categories:

| Status | Meaning | Requirements |
|--------|---------|--------------|
| <span style={{ backgroundColor: "#22c55e", color: "white", padding: "2px 8px", borderRadius: "4px", fontSize: "12px", fontWeight: "600" }}>CONFORMANT</span> | Product satisfies AARM specification requirements | Complete [testing protocol](/conformance/testing), verify R1-R7 (Core) or R1-R9 (Extended) |
| <span style={{ backgroundColor: "#3b82f6", color: "white", padding: "2px 8px", borderRadius: "4px", fontSize: "12px", fontWeight: "600" }}>ALIGNED</span> | Building in the same problem space | Working on runtime action security, self-declared |

## How to Get Listed

### AARM Conformant

1. Satisfy all MUST requirements (R1-R7) as defined in the [Conformance Requirements](/conformance/requirements)
2. Complete the [Testing Protocol](/conformance/testing) and provide evidence
3. Submit verification results via [GitHub](https://github.com/aarm-spec/aarm/issues)

### AARM Aligned

1. Your product addresses one or more aspects of the AI runtime security problem
2. Submit a brief description of what you're building
3. No conformance testing required
